YouTube Bot — a youtube bot workflow & analytics assistant

YouTube Bot is a ToS-compliant, human-in-the-loop toolkit that helps creators and teams manage channel workflows at scale—covering monitoring, scheduling assistance, comment triage queues, thumbnail/title experiments, and lightweight analytics dashboards.

Built for productivity and safe experimentation. No automated likes, comments, subscriptions, or views—ever.

View on GitHub LCP <2.5s · CLS <0.1 · INP <200ms

Features

Monitoring

Track channel/videos, new uploads, and comment activity via safe integrations (e.g., YouTube Data API) with configurable polling windows.

Scheduling Assistance

Plan titles, descriptions, and release cadence. Export schedules for manual review and publishing—humans approve before anything goes live.

Comment Triage

Queue, prioritize, and label comments for faster responses. No auto-commenting—this is a reviewer workflow to keep you in control.

Thumbnail & Title Experiments

Document A/B-style experiments and compare metrics. Experiments are guidance-only and executed manually to stay ToS-compliant.

Analytics Dashboards

Lightweight views for impressions, CTR, and retention summaries to inform editorial decisions without heavy dependencies.

Use Cases & Benefits

Solo Creator Productivity

Reduce context switching and keep everything from ideas to triage in one place.

Team Workflows

Share checklists, assign priorities, and keep a predictable release cadence.

SEO Insights

Spot trends in titles/thumbnails and track how changes align with discovery.

Content Optimization

Run safe, manual experiments and learn quickly—without risking your channel.

Installation

Prerequisites

  • Node.js 18+ and Git
  • (Optional) YouTube Data API key stored in .env

Clone & Setup

git clone https://github.com/Appilot123/youtube-bot
cd youtube-bot
npm install
# or: pnpm install | yarn install

Run Locally

npm run dev
# open http://localhost:5173 (Vite) or the printed URL

Build

npm run build
# output in ./dist – deploy to GitHub Pages

FAQ

Does this automate likes, comments, subscriptions, or views?
No. The project explicitly avoids any features that would violate YouTube Terms of Service. All workflows are human-in-the-loop.
Where does the data come from?
Primarily from the YouTube Data API and exported reports that you provide. Ensure you respect API quotas and applicable policies.
How do I deploy to GitHub Pages?
Build the project (npm run build) and publish the dist folder to the gh-pages branch, or enable Pages from the Settings → Pages menu.
Is there a roadmap or issue tracker?
Yes—use GitHub Issues to file bugs or vote on features.
